# 项目理解指南 4 — 族谱与成员管理

## 0. 前言

前三章建好了地基：数据库结构（指南2）和用户认证（指南3）。本章讲系统的核心业务功能：**族谱和成员的增删改查（CRUD）**，以及对应的前端界面。

CRUD 是所有数据系统最基础的四种操作：
- **C**reate（创建）：新建族谱、新建成员
- **R**ead（读取）：列出族谱、搜索成员
- **U**pdate（更新）：编辑成员信息
- **D**elete（删除）：删除族谱、删除成员

读完本章，你应当能够：
1. 解释 RESTful API 的设计原则（URL 对应资源，HTTP 方法对应操作）

3. 描述「分页」和「模糊搜索」的工作原理
4. 理解前端页面中状态管理（useState）和数据获取（useEffect）的配合方式
5. 解释「成员表单」如何复用于新建和编辑两种场景

## 4. 基础知识铺垫

### 4.1 什么是 RESTful API 设计

REST（Representational State Transfer）是 API 设计的一种规范。核心思想是：**URL 代表资源，HTTP 方法代表对资源的操作**。

本项目遵循的规范：

| URL | HTTP 方法 | 操作 |
|-----|---------|------|
| `/api/genealogies` | GET | 获取族谱列表 |
| `/api/genealogies` | POST | 创建新族谱 |
| `/api/genealogies/abc123` | GET | 获取 ID 为 abc123 的族谱 |
| `/api/genealogies/abc123` | PATCH | 更新 ID 为 abc123 的族谱 |
| `/api/genealogies/abc123` | DELETE | 删除 ID 为 abc123 的族谱 |
| `/api/members?genealogyId=abc&page=1&q=张` | GET | 搜索族谱中的成员 |
| `/api/members` | POST | 创建新成员 |
| `/api/members/def456` | PATCH | 更新 ID 为 def456 的成员 |

HTTP 方法的语义区别：
- **GET**：获取数据，不修改任何东西（幂等，反复调用结果相同）
- **POST**：创建新资源（非幂等，重复调用会创建多个）
- **PATCH**：部分更新资源（只传需要修改的字段）
- **DELETE**：删除资源

### 4.2 什么是「分页」

当数据库有 10 万条成员记录时，一次性返回全部数据会：
1. 占用大量内存（服务器和浏览器都撑不住）
2. 网络传输时间很长
3. 浏览器渲染 10 万行 DOM 极其卡顿

「分页」把结果分成若干页，每次只返回一页（本项目每页 20 条）。前端通过 `?page=1&pageSize=20` 这样的参数告诉后端要第几页：

```typescript
// 计算跳过多少条（第2页跳过前20条）
const skip = (page - 1) * pageSize;  // skip = 20
const take = pageSize;                 // take = 20

prisma.member.findMany({ skip, take, where: {...} })
```

Prisma 的 `skip` 和 `take` 对应 SQL 的 `OFFSET` 和 `LIMIT`：

```sql
SELECT * FROM members WHERE ... LIMIT 20 OFFSET 20;
```

### 4.3 什么是模糊搜索

精确搜索要求输入和数据库值完全匹配。模糊搜索允许部分匹配：搜索「张」能找到「张三」「张大山」「张小红」。

本项目使用 Prisma 的 `contains` 运算符：

```typescript
where: {
  name: { contains: "张", mode: "insensitive" }
}
```

对应 SQL：
```sql
WHERE name ILIKE '%张%'
```

`ILIKE` 是 PostgreSQL 的大小写不敏感的 LIKE 查询。`mode: "insensitive"` 就是让 Prisma 生成 `ILIKE` 而不是 `LIKE`。

### 4.4 什么是「权限控制」

权限控制确保用户只能操作自己有权限的数据。本项目的规则：
- 族谱**创建者**（owner）可以做任何操作
- 被邀请的**协作者**也可以查看和修改成员

`canAccessGenealogy` 函数实现这个检查：

```typescript
async function canAccessGenealogy(genealogyId: string, userId: string) {
  const g = await prisma.genealogy.findUnique({
    where: { id: genealogyId },
    include: { collaborators: { where: { userId } } },
  });
  if (!g) return false;
  return g.ownerId === userId || g.collaborators.length > 0;
}
```

逻辑：查出族谱的 `ownerId` 和该用户是否在协作者列表里，满足其一返回 `true`。

### 4.5 什么是 React 的 `useState` 和 `useEffect`

**`useState`** 是 React 的状态管理钩子（Hook）。当一个值改变时需要重新渲染 UI，就用 `useState` 存它：

```typescript
const [members, setMembers] = useState<Member[]>([]);
// members：当前值
// setMembers：修改函数（调用后 React 自动重渲染）
```

**`useEffect`** 是副作用钩子。「副作用」指和渲染无关但需要执行的操作，如：获取数据、操作 DOM、设置定时器。

```typescript
useEffect(() => {
  // 这里的代码在组件渲染后执行
  fetch("/api/members?...").then(res => res.json()).then(data => setMembers(data.members));
}, [page, query]);  // 依赖数组：page 或 query 改变时重新执行
```

依赖数组的规则：
- `[]`（空数组）：只在组件首次挂载时执行一次
- `[a, b]`：a 或 b 的值变化时重新执行
- 不传：每次渲染都执行（通常不应该这样）

### 4.6 什么是 `useCallback`

`useCallback` 是 React 的另一个钩子，用来「缓存」一个函数，避免它每次渲染都被重新创建。

```typescript
const load = useCallback(async () => {
  // 获取数据的逻辑
}, [genealogyId, page, query]);  // 只有这些依赖变化时才创建新函数

useEffect(() => { load(); }, [load]);  // load 变化时重新执行
```

为什么需要这样做？因为如果 `load` 不被 `useCallback` 包裹，它每次渲染都是一个新函数，`useEffect` 的依赖 `[load]` 每次都变化，导致无限循环请求数据。

## 5. 逐步代码讲解

### 5.1 步骤 1：`app/api/genealogies/route.ts` — 族谱列表和创建 【开发者甲】

#### A. 动机

这个文件处理两种请求：
- `GET /api/genealogies`：返回当前用户有权访问的族谱列表（自建的 + 被邀请的）
- `POST /api/genealogies`：创建新族谱

#### B. 完整代码

*文件 `app/api/genealogies/route.ts`：*

```typescript
import { NextRequest, NextResponse } from "next/server";
import { auth } from "@/lib/auth";
import { prisma } from "@/lib/db";

// 获取当前用户的族谱列表（自建 + 受邀）
export async function GET() {
  const session = await auth();
  if (!session) return NextResponse.json({ error: "未登录" }, { status: 401 });

  const genealogies = await prisma.genealogy.findMany({
    where: {
      OR: [
        { ownerId: session.user!.id },
        { collaborators: { some: { userId: session.user!.id } } },
      ],
    },
    include: {
      owner: { select: { username: true } },
      _count: { select: { members: true } },
    },
    orderBy: { createdAt: "desc" },
  });

  return NextResponse.json(genealogies);
}

// 创建族谱
export async function POST(req: NextRequest) {
  const session = await auth();
  if (!session) return NextResponse.json({ error: "未登录" }, { status: 401 });

  const { name, surname, createdYear } = await req.json();
  if (!name || !surname) {
    return NextResponse.json({ error: "谱名和姓氏为必填项" }, { status: 400 });
  }

  const genealogy = await prisma.genealogy.create({
    data: {
      name,
      surname,
      createdYear: createdYear ?? new Date().getFullYear(),
      ownerId: session.user!.id as string,
    },
  });
  return NextResponse.json(genealogy, { status: 201 });
}
```

#### C. 这段代码做了什么

**权限检查：认证模式**

```typescript
const session = await auth();
if (!session) return NextResponse.json({ error: "未登录" }, { status: 401 });
```

每个 API 接口的第一行都是这个检查。`auth()` 读取 Cookie 里的 JWT，返回 Session 对象（未登录时返回 `null`）。

401 是 HTTP 状态码「未认证」（Unauthorized）的标准值。

**复杂的 `OR` 查询**

```typescript
where: {
  OR: [
    { ownerId: session.user!.id },
    { collaborators: { some: { userId: session.user!.id } } },
  ],
}
```

这两个条件的含义：
- `ownerId === 当前用户ID`：这个族谱是我建的
- `collaborators.some(c => c.userId === 当前用户ID)`：有一条协作记录指向我

`some` 是 Prisma 的关系过滤，意思是「至少有一条关联记录满足条件」。对应 SQL：

```sql
WHERE owner_id = 'user1'
  OR EXISTS (
    SELECT 1 FROM genealogy_collaborators
    WHERE genealogy_id = genealogies.id AND user_id = 'user1'
  )
```

**`include` 关联查询**

```typescript
include: {
  owner: { select: { username: true } },
  _count: { select: { members: true } },
}
```

- `owner: { select: { username: true } }`：顺便查出创建者的用户名（只取 `username` 字段，不取密码哈希）
- `_count: { select: { members: true } }`：统计每个族谱的成员数量，对应 SQL 的 `COUNT(*)`

> **反例对比**：如果不用 `include`，只查族谱基本信息，前端要显示成员数量时需要再发一次请求，或者一次 N 个族谱需要 N 次请求（N+1 问题）。用 `include` 一次查询搞定，效率高很多。

### 5.2 步骤 2：`app/api/members/route.ts` — 成员列表、创建 【开发者甲】

#### A. 动机

这是成员管理的核心接口，需要处理：分页、模糊搜索、权限验证，以及创建时同时建立亲子关系。

#### B. 完整代码

*文件 `app/api/members/route.ts`：*

```typescript
import { NextRequest, NextResponse } from "next/server";
import { auth } from "@/lib/auth";
import { prisma } from "@/lib/db";

async function canAccessGenealogy(genealogyId: string, userId: string) {
  const g = await prisma.genealogy.findUnique({
    where: { id: genealogyId },
    include: { collaborators: { where: { userId } } },
  });
  if (!g) return false;
  return g.ownerId === userId || g.collaborators.length > 0;
}

// 列出族谱成员（支持姓名模糊搜索）
export async function GET(req: NextRequest) {
  const session = await auth();
  if (!session) return NextResponse.json({ error: "未登录" }, { status: 401 });

  const genealogyId = req.nextUrl.searchParams.get("genealogyId");
  const q = req.nextUrl.searchParams.get("q") ?? "";
  const page = parseInt(req.nextUrl.searchParams.get("page") ?? "1");
  const pageSize = parseInt(req.nextUrl.searchParams.get("pageSize") ?? "20");

  if (!genealogyId) return NextResponse.json({ error: "缺少 genealogyId" }, { status: 400 });
  if (!(await canAccessGenealogy(genealogyId, session.user!.id))) {
    return NextResponse.json({ error: "无权访问" }, { status: 403 });
  }

  const where = {
    genealogyId,
    ...(q ? { name: { contains: q, mode: "insensitive" as const } } : {}),
  };

  const [members, total] = await Promise.all([
    prisma.member.findMany({
      where,
      orderBy: [{ generation: "asc" }, { birthYear: "asc" }],
      skip: (page - 1) * pageSize,
      take: pageSize,
    }),
    prisma.member.count({ where }),
  ]);

  return NextResponse.json({ members, total, page, pageSize });
}

// 创建成员
export async function POST(req: NextRequest) {
  const session = await auth();
  if (!session) return NextResponse.json({ error: "未登录" }, { status: 401 });

  const body = await req.json();
  const { genealogyId, name, gender, birthYear, deathYear, bio, generation, parentIds, spouseIds } = body;

  if (!genealogyId || !name || !gender) {
    return NextResponse.json({ error: "缺少必填字段" }, { status: 400 });
  }
  if (!(await canAccessGenealogy(genealogyId, session.user!.id))) {
    return NextResponse.json({ error: "无权操作" }, { status: 403 });
  }
  if (deathYear && birthYear && deathYear <= birthYear) {
    return NextResponse.json({ error: "逝世年份必须晚于出生年份" }, { status: 400 });
  }

  const member = await prisma.member.create({
    data: {
      genealogyId,
      name,
      gender,
      birthYear: birthYear ?? null,
      deathYear: deathYear ?? null,
      bio: bio ?? null,
      generation: generation ?? null,
    },
  });

  // 建立亲子关系
  if (parentIds?.length) {
    await prisma.parentChild.createMany({
      data: parentIds.map((pid: string) => ({ parentId: pid, childId: member.id })),
      skipDuplicates: true,
    });
  }

  // 建立婚姻关系
  if (spouseIds?.length) {
    await prisma.marriage.createMany({
      data: spouseIds.map((sid: string) => ({ member1Id: member.id, member2Id: sid })),
      skipDuplicates: true,
    });
  }

  return NextResponse.json(member, { status: 201 });
}
```

#### C. 这段代码做了什么

**URL 查询参数解析**

```typescript
const genealogyId = req.nextUrl.searchParams.get("genealogyId");
const q = req.nextUrl.searchParams.get("q") ?? "";
const page = parseInt(req.nextUrl.searchParams.get("page") ?? "1");
```

`req.nextUrl.searchParams` 解析 URL 里 `?` 后面的参数。`?? ""` 表示「如果为 null 就用空字符串」（这是 TypeScript 的「空值合并运算符」）。

**同时执行两个查询**

```typescript
const [members, total] = await Promise.all([
  prisma.member.findMany({ where, skip, take }),
  prisma.member.count({ where }),
]);
```

`Promise.all` 让两个 Prisma 查询**并发执行**，而不是先查完列表再查总数（那样需要两倍时间）。结果用数组解构赋值。

**动态构建 `where` 条件**

```typescript
const where = {
  genealogyId,
  ...(q ? { name: { contains: q, mode: "insensitive" as const } } : {}),
};
```

这是 TypeScript 的「展开运算符」（`...`）和三元表达式组合：
- 如果 `q` 不为空：加入姓名模糊搜索条件
- 如果 `q` 为空：什么都不加（展开空对象 `{}`）

**`skipDuplicates: true`**

```typescript
await prisma.parentChild.createMany({
  data: parentIds.map((pid) => ({ parentId: pid, childId: member.id })),
  skipDuplicates: true,
});
```

`createMany` 批量插入多条记录。`skipDuplicates: true` 表示：如果记录已存在（违反主键约束），不报错，跳过这条。避免重复提交时崩溃。

### 5.3 步骤 3：`app/(main)/genealogies/[id]/members/page.tsx` — 成员管理页 【开发者乙】

这是最复杂的前端页面，包含了分页、搜索、CRUD 操作和 ID 复制功能。分几块讲解：

#### A. 动机

成员管理页需要提供完整的成员管理功能，同时为演示需要提供显示和复制成员 ID 的功能（ID 在祖先查询和亲缘查询时需要输入）。

#### B. `CopyIdButton` 组件

```typescript
function CopyIdButton({ id }: { id: string }) {
  const [copied, setCopied] = useState(false);

  function handleCopy(e: React.MouseEvent) {
    e.stopPropagation();
    navigator.clipboard.writeText(id).then(() => {
      setCopied(true);
      setTimeout(() => setCopied(false), 1500);
    });
  }

  return (
    <button onClick={handleCopy} title="复制完整 ID" ...>
      {copied
        ? <Check size={13} className="text-emerald-500" />
        : <Copy size={13} className="text-zinc-400" />}
    </button>
  );
}
```

**`navigator.clipboard.writeText(id)`**：调用浏览器的剪贴板 API 复制文本，返回 Promise。

**`e.stopPropagation()`**：阻止点击事件「冒泡」到父元素（如果不阻止，点复制按钮可能触发父元素的点击事件）。

**临时提示效果**：`setCopied(true)` 变成绿色对勾，1500ms 后用 `setTimeout` 回退到复制图标。

#### C. `IdModal` 组件（弹窗）

```typescript
function IdModal({ member, onClose }: { member: Member; onClose: () => void }) {
  return (
    <Modal open onClose={onClose} title="成员 ID">
      <div>
        <p>{member.name}</p>
        <div className="flex items-center gap-2 ...">
          <code className="flex-1 text-sm font-mono ... select-all">{member.id}</code>
          <button onClick={handleCopy}>
            {copied ? <Check /> : <Copy />}
          </button>
        </div>
        <p>点击 ID 文本可全选，点击复制按钮可一键复制</p>
      </div>
    </Modal>
  );
}
```

关键点：
- `<code>` 标签语义上表示代码/技术内容（在这里是 ID 字符串），字体会切换为等宽字体（`font-mono`）
- `select-all`：这是一个自定义 Tailwind 类，对应 CSS `user-select: all`，点击文本自动全选

#### D. 主页面的状态设计

```typescript

const [total, setTotal] = useState(0);                   // 总成员数

const [inputQ, setInputQ] = useState("");                // 搜索框正在输入的值
const [loading, setLoading] = useState(true);            // 加载状态
const [showForm, setShowForm] = useState(false);         // 是否显示表单弹窗
const [editing, setEditing] = useState<Member | null>(null); // 正在编辑的成员
const [viewingId, setViewingId] = useState<Member | null>(null); // 正在查看ID的成员
```

注意 `query` 和 `inputQ` 是两个状态：
- `inputQ`：用户实时在搜索框输入的内容（每按一个键更新）
- `query`：实际触发搜索的关键词（只有按下搜索按钮时才更新）

这样设计是为了「按下搜索才触发请求」，而不是「每输入一个字就请求一次」。

#### E. 完整代码

*文件 `app/(main)/genealogies/[id]/members/page.tsx`（核心部分）：*

```typescript

import { useParams } from "next/navigation";
// ... 其他 import

export default function MembersPage() {
  const { id: genealogyId } = useParams<{ id: string }>();

    setLoading(false);
  }, [genealogyId, page, query]);

  // ... 其他处理函数和 JSX
}
```

**`useParams`**：从 URL 动态段中读取参数。对于路径 `/genealogies/abc123/members`，`useParams().id` 就是 `"abc123"`。

### 5.4 步骤 4：`components/forms/member-form.tsx` — 成员表单 【开发者乙】

#### A. 动机

「新建成员」和「编辑成员」的表单几乎完全相同，写两份会造成重复。用一个可复用的 `MemberForm` 组件，通过 `initial` prop 判断是新建还是编辑：
- `initial` 为 `null`：新建模式，表单为空，提交时调用 POST
- `initial` 有值：编辑模式，表单预填数据，提交时调用 PATCH

#### B. 关键代码

*文件 `components/forms/member-form.tsx`（核心逻辑）：*

```typescript
const isEdit = !!initial?.id;

// 表单状态初始化：编辑模式从 initial 读，新建模式用默认值
const [form, setForm] = useState({
  name: initial?.name ?? "",
  gender: initial?.gender ?? "M",
  birthYear: initial?.birthYear ? String(initial.birthYear) : "",
  ...
});

async function handleSubmit(e: React.FormEvent) {
  e.preventDefault();
  // 根据是否是编辑模式决定 URL 和方法
  const url = isEdit ? `/api/members/${initial!.id}` : "/api/members";
  const method = isEdit ? "PATCH" : "POST";

  const res = await fetch(url, {
    method,
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(payload),
  });
  if (!res.ok) { setError(data.error); return; }
  onSuccess();  // 通知父组件成功（关闭弹窗、刷新列表）
}
```

**`??` 运算符**：`initial?.name ?? ""` 意思是「如果 `initial.name` 存在就用它，否则用空字符串」。

**`?.` 可选链**：`initial?.id` 意思是「如果 `initial` 不为 null/undefined，就访问它的 `id`，否则返回 `undefined`」。

**父 ID 输入方式**：

```typescript
onKeyDown={(e) => {
  if (e.key === "Enter") {
    e.preventDefault();
    const val = form.parentInput.trim();
    if (val && !form.parentIds.includes(val)) {
      setForm((f) => ({ ...f, parentIds: [...f.parentIds, val], parentInput: "" }));
    }
  }
}}
```

用户在父母ID输入框里输入一个ID后按回车，把这个ID加入 `parentIds` 数组，输入框清空。这是「标签输入」的交互模式（类似邮件收件人输入）。

## 7. 本章小结

| 概念 | 一句话记忆 |
|------|-----------|
| RESTful API | URL 代表资源，HTTP 方法代表操作 |
| 401 / 403 / 404 | 未认证 / 无权限 / 资源不存在 的 HTTP 状态码 |
| `Promise.all` | 并发执行多个异步操作，全部完成后统一返回 |
| `skip` + `take` | Prisma 分页：跳过前 N 条，取 M 条 |
| `contains` + `mode: "insensitive"` | Prisma 模糊搜索，不区分大小写 |
| `useState` | React 状态钩子，值改变时触发重渲染 |
| `useEffect` | React 副作用钩子，处理数据获取等操作 |
| `useCallback` | 缓存函数引用，避免 useEffect 无限循环 |
| 表单复用 | 通过 `initial` prop 判断新建/编辑模式 |

下一章（指南5）将讲解系统最核心的数据库算法部分：用递归 CTE 查询祖先、用 BFS 算法找亲缘路径，以及用 ECharts 可视化这些结果。
