# 数据库导出指南（pg_dump）

本指南说明如何使用 PostgreSQL 官方工具 `pg_dump` 将整个 `genealogyms` 数据库导出为文件，用于实验提交（满足第4条「提交导出文件」的要求）。

---

## 前置条件

- PostgreSQL 18 服务正在运行
- 数据库 `genealogyms` 中已有数据（执行过 `npm run db:seed`）

---

## 方法一：导出为自定义格式（推荐，文件小）

在终端（不是 psql，是普通 Terminal）执行：

```bash
pg_dump \
  -U postgres \
  -d genealogyms \
  -F c \
  -f /Users/arielle/Desktop/GenealogyMS/SUBMISSION/genealogyms_backup.dump
```

参数说明：
- `-U postgres`：使用 postgres 用户连接
- `-d genealogyms`：导出 genealogyms 数据库
- `-F c`：自定义格式（Custom），压缩存储，文件最小
- `-f 路径`：输出文件路径

执行后系统会提示输入密码，输入你的 PostgreSQL 密码（`040604`）。

完成后验证文件：
```bash
ls -lh /Users/arielle/Desktop/GenealogyMS/SUBMISSION/genealogyms_backup.dump
# 应显示文件大小（通常 5-50 MB，取决于数据量）
```

---

## 方法二：导出为 SQL 文本格式（可直接阅读）

```bash
pg_dump \
  -U postgres \
  -d genealogyms \
  -F p \
  --no-owner \
  --no-acl \
  -f /Users/arielle/Desktop/GenealogyMS/SUBMISSION/genealogyms_backup.sql
```

参数说明：
- `-F p`：纯文本格式（Plain SQL），可以直接用文本编辑器打开查看
- `--no-owner`：不包含 OWNER 语句（迁移到其他机器时不报错）
- `--no-acl`：不包含权限语句

SQL 格式文件较大（未压缩），数据量大时可能有几百 MB，但内容是可读的 SQL 语句。

---

## 方法三：仅导出结构（不含数据，用于展示表结构）

```bash
pg_dump \
  -U postgres \
  -d genealogyms \
  -F p \
  --schema-only \
  --no-owner \
  -f /Users/arielle/Desktop/GenealogyMS/SUBMISSION/genealogyms_schema_only.sql
```

`--schema-only`：只导出建表语句和索引，不导出数据行。文件很小，适合展示数据库结构设计。

---

## 推荐提交组合

建议同时提交以下两个文件：

| 文件 | 格式 | 用途 |
|------|------|------|
| `genealogyms_backup.dump` | 自定义格式（二进制压缩） | 完整数据库备份，可用 `pg_restore` 还原 |
| `genealogyms_schema_only.sql` | 纯文本 SQL | 方便老师阅读数据库结构 |

## 验证导出文件（可选）

导出完成后，可以查看备份文件包含的内容：

```bash
# 列出自定义格式备份文件的目录（不解压，只看包含哪些内容）
pg_restore --list /Users/arielle/Desktop/GenealogyMS/SUBMISSION/genealogyms_backup.dump | head -30
```

输出应包含类似内容：
```
;
; Archive created at ...
; dbname: genealogyms
; ...
;
; Selected TOC Entries:
;
3; 2615 2200 SCHEMA - public postgres
...
TABLE DATA public users postgres
TABLE DATA public genealogies postgres
TABLE DATA public members postgres
...
```

## 还原方法（供老师验证使用）

如果老师需要在自己的机器上还原这个数据库：

```bash
# 1. 创建数据库
createdb -U postgres genealogyms_restored

# 2. 还原（使用自定义格式的 .dump 文件）
pg_restore \
  -U postgres \
  -d genealogyms_restored \
  --no-owner \
  /path/to/genealogyms_backup.dump
```

## 常见问题

**Q：执行 pg_dump 提示 "command not found"？**

PostgreSQL 安装包通常把工具安装到 `/Library/PostgreSQL/18/bin/`，需要添加到 PATH：
```bash
export PATH="/Library/PostgreSQL/18/bin:$PATH"
# 或者用完整路径执行：
/Library/PostgreSQL/18/bin/pg_dump -U postgres -d genealogyms -F c -f /Users/arielle/Desktop/GenealogyMS/SUBMISSION/genealogyms_backup.dump
```

**Q：提示权限拒绝（Permission denied）？**

确保输出目录有写入权限：
```bash
ls -la /Users/arielle/Desktop/GenealogyMS/SUBMISSION/
# 如果没有写权限，执行：
chmod 755 /Users/arielle/Desktop/GenealogyMS/SUBMISSION/
```

**Q：文件很大（>100MB）怎么办？**

自定义格式（`-F c`）会自动压缩，通常比实际数据小很多。如果依然很大，可以只导出结构（`--schema-only`）并附上一个说明文件解释数据规模。
